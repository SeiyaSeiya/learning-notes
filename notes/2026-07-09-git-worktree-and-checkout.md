# git worktree と checkout の違い

- 学んだきっかけ: `gas-voc-extract`（GASプロジェクト）でClaude Codeにドキュメント作成を依頼した際、作業ディレクトリが `.claude/worktrees/xxx` という見慣れない場所になっていたことから調査。

## 1. 今回の作業環境の実態

- 作業していたディレクトリは、単純なファイルコピーではなく **git worktree**（同じリポジトリの別ブランチを別ディレクトリに展開したもの）だった。
- 本来のディレクトリ（`main`ブランチ）と、そのworktree（`claude/xxx`ブランチ）は、**同じ `.git`（履歴データ）を共有**している。
- worktree側への変更は本来のディレクトリ（`main`）に一切影響しない。取り込むには、ブランチをpush→PR作成→マージ、または `git merge` が必要。
- これはClaude Code自体の標準動作ではなく、セッションを起動している環境（ハーネス）側が、安全のためタスクごとに専用worktree＋ブランチを自動作成する設定になっていたため。素の `claude` コマンドでは、通常は今いるディレクトリに直接書き込む。

## 2. `git branch` の `+` マークの意味

```
+ claude/gas-project-overview-docs-df0ae9
* main
```

- `*` … **今いるディレクトリ**でチェックアウト中のブランチ
- `+` … **別のworktree（別ディレクトリ）**でチェックアウト中のブランチ
- 記号なし … どこにもチェックアウトされていないブランチ
- この記号は、複数worktreeを使っている時だけ意味を持つ表示。

## 3. 「チェックアウト」の意味

- git は内部にコミット（スナップショット）を記録しているが、それ自体は直接見えない・編集できない。
- 「チェックアウト」とは、**指定したブランチのスナップショットの中身を、実際にディスク上のファイルとして展開すること**。
- 1つのディレクトリには1ブランチ分のチェックアウトしか同時にできない（ファイルが競合するため）。

## 4. `git checkout`/`git switch` と `git worktree add` の違い

| コマンド | 動作 |
|---|---|
| `git checkout -b newbranch` / `git switch -c newbranch` | 新ブランチを作成し、**今のディレクトリの中身をそのブランチに切り替える**（元のブランチの内容は消える） |
| `git worktree add <path> -b newbranch` | 新ブランチを作成するが、**今のディレクトリは切り替えない**。別ディレクトリを新設し、そこにだけ新ブランチをチェックアウトする |

- 通常の `checkout`/`switch` は「1つの場所でブランチを入れ替える」動作。
- `worktree add` は「場所を増やして両方同時に存在させる」動作。
- 今回使われたのは後者だったため、本来のディレクトリは終始 `main` のままで、一度も切り替わっていなかった。

## 5. 派生した気づき：セッション間の知識の引き継ぎ

- Claude Codeの自動メモ機能は作業中のプロジェクト（ディレクトリ）に紐づく仕組みのため、別ディレクトリで新しいセッションを開くと自動的には引き継がれない。
- 確実に引き継ぐには、内容を**ファイルとして書き出しておく**のが良い（ファイルはセッションをまたいで残るため）。このリポジトリ（`learning-notes`）自体がその実践。
