# VS CodeでのClaude Code（ターミナル版）運用と効率化

## 概要
2026-07-11のメモでターミナル版（CLI）の優位性を確認した。本日は、エンジニアの開発ワークフローとして、ターミナル版をVS Codeの統合ターミナルで運用するメリットと、具体的な快適化の設定・操作方法についてまとめる。

## VS Code統合ターミナルで運用するメリット
ターミナル版（CLI）をVS Code内で動かすことで、開発効率が大きく向上する。

* **視覚的なスイッチングコストの排除**
  * 上部にエディタ、下部にターミナル（Claude Code）を配置。AIによるファイル修正がリアルタイムでエディタに反映されるため、画面を切り替える手間がない。
* **カレントディレクトリのコンテキスト共有**
  * 開いているプロジェクトのディレクトリ構造をAIが把握しているため、「このディレクトリのテストを実行して」「package.jsonを元に更新して」などの指示がシームレスに伝わる。
* **既存ツールとの親和性**
  * 同じターミナル内で `git diff` や `git commit` などのコマンド操作をそのまま行える。将来的にはシェルスクリプトやCI/CDへの組み込みも容易。

## ターミナル操作のショートカット設定 (Mac環境)
AIとのやり取りをマウスレスで高速化するためのVS Codeショートカット設定。

### 基本の画面切り替え
* **`Cmd + Shift + [ または ]`**
  * エディタのタブを左右に切り替える（ブラウザと同じ感覚）。
* **`Ctrl + 1`**
  * ターミナルからエディタ（コード編集画面）へフォーカスを戻す。
* **`Ctrl + \`` (バッククォート)**
  * エディタ（コード編集画面）からターミナルへフォーカスを切り替える。
* **`Cmd + J`**
  * ターミナルの表示/非表示を切り替える。

### ターミナル最大化の推奨設定（トグル化）
AIが大量のログやソースコードの差分を出力した際、1キーで最大化/復元できるようにカスタマイズを行った。

**【設定手順】**
1. キーボードショートカット設定を開く（`Cmd + K` -> `Cmd + S`）。
2. `View: Toggle Maximized Panel`（パネルの最大化の切り替え）を検索。
3. **`Cmd` + `Shift` + `J`** に割り当てる。

これにより、ログが長くなった瞬間に `Cmd + Shift + J` で全画面表示にし、確認後は同じキーですぐにエディタ画面へ戻れる。

### ターミナルタブの削除は `exit` コマンドで
不要になったターミナルタブは、ゴミ箱アイコンやショートカットでウィンドウごと閉じるのではなく、タブ内で `exit` コマンドを実行して終了させるのが基本。シェルプロセスを正しく終了させてからタブが閉じるため、実行中のプロセスを強制終了する形にならず安全。

## CLI版での画像（スクリーンショット）の渡し方
ターミナル版のClaude Codeでも、デスクトップ版と同様に画像を認識させることが可能（Mac環境）。

### 方法1：クリップボードから直接貼り付ける（推奨）
1. `Cmd + Ctrl + Shift + 4` で画面の範囲選択をしてスクリーンショットを撮影（クリップボードに保存される）。
2. Claude Codeを実行中のターミナルにフォーカスを当て、`Cmd + V` を押す。
3. プロンプトに `[Image #1]` のように表示されれば成功。

### 方法2：ファイルに保存してドラッグ＆ドロップ
1. `Cmd + Shift + 4` 等でスクリーンショットをデスクトップなどにファイル保存。
2. 保存した画像ファイルを、Claude Codeが起動しているターミナル画面へドラッグ＆ドロップ。
3. ターミナル上にファイルパス（例: `/Users/username/Desktop/hoge.png`）が自动で入力される。

**⚠️ 重要（共通の注意点）**
画像（`[Image #1]` やファイルパス）を貼り付けただけではAIは処理を実行できない。必ず**「この画像のエラー原因を教えて」「このUIの改善点を教えて」などのテキスト指示を併記して**から `Enter` を押すこと。

## Claude Code CLI 基本操作の再整理
- 学んだきっかけ: `gas-feedback-generate` プロジェクトでCLI版Claude Codeを初導入した際の質疑応答。

### よく使うスラッシュコマンド
| コマンド | 用途 |
|---|---|
| `/help` | コマンド一覧表示 |
| `/clear` | 新しい会話を開始（プロジェクト設定は保持） |
| `/config` | テーマ・モデルなどの設定変更 |
| `/model` | 使用モデルの切り替え |
| `/memory` | CLAUDE.mdなど設定ファイルの確認・編集 |
| `/resume` | 過去の会話を再開 |

### 権限モード（`Shift + Tab` で切り替え）
- **Manual**: 編集・コマンド実行のたびに確認（導入初期はこれが安全）。
- **Accept edits**: ファイル編集は自動承認、外部コマンド実行は都度確認。
- **Auto**: 信頼できる操作はほぼ自動実行。

### CLAUDE.mdの役割（再確認）
プロジェクトルートや `.claude/` に置くと、セッション開始時に自動で読み込まれ、プロジェクト固有の前提・注意点をAIに引き継げる。`gas-project` のように複数の独立リポジトリをまとめて管理している場合、「全体共通ルール（親ディレクトリ）」と「プロジェクト固有の壊れやすい前提（各サブディレクトリ）」を分けて書く構成が有効。

## 画像ペースト時のキャッシュフォルダについて
- ペーストした画像は `~/.claude/image-cache/<セッションID>/` に一時的にPNGファイルとして保存される。プロンプト上に表示される `[Image: source: /Users/.../image-cache/.../N.png]` という表記はエラーではなく、正常な参照情報（保存場所を示しているだけ）。
- 不要になれば `rm -rf ~/.claude/image-cache/*` で削除して問題ない。貼り付け時点で画像の内容はすでにAI側に読み込まれているため、後からキャッシュを消しても直前までの会話には影響しない。
- 画像ペースト時のショートカットキーは環境依存の可能性がある点に注意。本セッションではCtrl+Vでの案内を受けたが、上記の別記録ではCmd+Vで成功した例もある。ターミナルアプリ（Terminal.app / iTerm2 / VS Code統合ターミナルなど）によって挙動が異なる可能性があるため、片方でうまくいかない場合はもう一方も試すとよい。
