# Vimの基本操作

## 概要
vimをほとんど使ったことがなかったので、基本操作を一通り整理した。普段はVS Code等のGUIエディタを使うが、SSHでのリモートサーバー接続時や、`git commit`（メッセージ引数なし実行時のデフォルトエディタ）、`crontab -e`など、ターミナルのみでファイル編集が必要な場面は今後も出てくるため、最低限の操作を押さえておく。

## 3つの基本モード
vimには複数のモードがあり、モードによってキー入力の意味が変わる。これが他のエディタと最も違う点で、最初につまずきやすい。

| モード | 入り方 | 役割 |
|---|---|---|
| ノーマルモード | 起動直後 / `Esc` | カーソル移動・コマンド実行（デフォルトの状態） |
| 挿入モード | ノーマルモードで `i`, `a`, `o` など | 実際に文字を入力する |
| ビジュアルモード | ノーマルモードで `v`, `V`, `Ctrl+v` | 範囲選択（コピー・削除・置換の対象を選ぶ） |
| コマンドラインモード | ノーマルモードで `:` | 保存・終了・検索置換などのコマンドを実行 |

**重要**: 操作に迷ったら、まず `Esc` を押してノーマルモードに戻る。

## 起動・保存・終了
| コマンド | 内容 |
|---|---|
| `vim ファイル名` | ファイルを開く（なければ新規作成） |
| `:w` | 保存 |
| `:q` | 終了（未保存の変更があると失敗する） |
| `:wq` または `ZZ` | 保存して終了 |
| `:q!` | 保存せず強制終了 |
| `:w ファイル名` | 別名で保存 |

## 挿入モードへの入り方（ノーマルモードから）
| コマンド | 内容 |
|---|---|
| `i` | カーソル位置の前に挿入 |
| `a` | カーソル位置の後に挿入 |
| `I` | 行頭に挿入 |
| `A` | 行末に挿入 |
| `o` | 下に新しい行を作って挿入 |
| `O` | 上に新しい行を作って挿入 |

## カーソル移動（ノーマルモード）
| コマンド | 内容 |
|---|---|
| `h` `j` `k` `l` | 左・下・上・右（矢印キーの代わり） |
| `w` / `b` | 次の単語の先頭へ / 前の単語の先頭へ |
| `0` / `$` | 行頭へ / 行末へ |
| `gg` / `G` | ファイルの先頭へ / 末尾へ |
| `:数字` または `数字G` | 指定した行へジャンプ（例: `:10` で10行目） |
| `Ctrl+f` / `Ctrl+b` | 1画面分下 / 上にスクロール |

## 編集操作
| コマンド | 内容 |
|---|---|
| `x` | カーソル位置の1文字を削除 |
| `dd` | 1行削除（カット） |
| `yy` | 1行コピー（ヤンク） |
| `p` / `P` | 貼り付け（カーソルの後 / 前） |
| `u` | 元に戻す（undo） |
| `Ctrl+r` | やり直し（redo） |
| `dw` | カーソル位置から単語末尾まで削除 |
| `数字` + コマンド | コマンドをその回数分繰り返す（例: `3dd` で3行削除） |

## 検索・置換
| コマンド | 内容 |
|---|---|
| `/検索語` + Enter | 前方検索、`n` で次、`N` で前 |
| `?検索語` + Enter | 後方検索 |
| `:%s/old/new/g` | ファイル全体で old を new に置換 |
| `:%s/old/new/gc` | 置換前に1件ずつ確認 |

## ビジュアルモードでの範囲選択
1. ノーマルモードで `v`（文字単位）または `V`（行単位）を押す。
2. カーソル移動で範囲を選択。
3. `d`（削除）、`y`（コピー）、`>`（インデント）などを実行。

## つまずきやすいポイント
- ノーマルモードで文字を打つと、入力ではなくコマンドとして解釈される（例: `dd` は削除コマンド）。まず `Esc` でノーマルモードに戻る癖をつける。
- `:q` で終了できない場合、変更が保存されていない可能性が高い。`:wq` で保存して終了するか、変更を破棄してよければ `:q!` を使う。

## このプロジェクトでの活用場面（学んだきっかけ）
- `git commit`をメッセージ引数なしで実行した際にデフォルトエディタとしてvimが開く環境があること、SSHでリモートサーバーに入った際にGUIエディタが使えないことを踏まえ、基礎だけでも把握しておくことにした。
- このリポジトリ自体はVS Code + Claude Code CLIで運用しており（[2026-07-13のメモ](2026-07-13-claude-code-cli-and-vscode-integration.md)参照）普段は困らないが、`crontab -e`やlaunchd関連の設定ファイル（[2026-07-14のメモ](2026-07-14-launchd-auto-commit.md)参照）をサーバー上で直接編集するような場面では、今後vimの操作が必要になる可能性がある。
